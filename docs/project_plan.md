# 🎯 Vibe Coding 專案式課程規劃

根據學生的學習目標，我設計了三個循序漸進的專案，核心理念是：**「用自然語言描述需求，讓 AI 幫你寫 code」**

---

## 📊 課程總覽

| 專案 | 難度 | 核心學習 | 對應學生需求 |
|------|------|----------|--------------|
| 個人記帳分析師 | ⭐ 入門 | 資料流基礎、Excel處理 | Python入門、Excel處理、視覺化 |
| 天氣數據追蹤站 | ⭐⭐ 中級 | API串接、自動化流程 | 爬蟲概念、資料分析 |
| AI 智慧報表產生器 | ⭐⭐⭐ 進階 | AI整合、完整系統架構 | AI/機器學習應用 |

---

## 🟢 專案一：個人記帳分析師

> **情境**：幫自己做一個能分析消費習慣的小工具

### 🔷 階段 1：資料處理小工具

**學習目標**
- 理解「輸入 → 處理 → 輸出」的資料流
- 學會用自然語言向 AI 描述需求

**Vibe Coding 練習**
```
💬 給 AI 的 Prompt 範例：
「我有一個 Excel 記帳檔案，欄位有日期、項目、金額、類別。
請幫我寫一個 Python 程式：
1. 讀取這個 Excel
2. 計算每個類別的總花費
3. 找出花最多錢的類別
4. 輸出成新的 Excel」
```

**資料流架構圖**
```
📁 Excel檔案 → 🐍 Python讀取 → 🔄 分類統計 → 📊 輸出結果
```

**重點觀念**
- 不用背語法，學會「描述你要什麼」
- 理解資料的「型態」（數字、文字、日期）
- 學會看 AI 給的程式碼「做了什麼」而非「怎麼寫」

---

### 🔷 階段 2：用 Python 畫圖

**學習目標**
- 理解「資料 → 圖表」的轉換邏輯
- 學會選擇適合的圖表類型

**Vibe Coding 練習**
```
💬 給 AI 的 Prompt 範例：
「根據上一步的類別統計結果，幫我：
1. 畫一個圓餅圖顯示各類別佔比
2. 畫一個長條圖顯示每月花費趨勢
3. 圖表要有中文標題，顏色要好看」
```

**資料流架構圖**
```
📊 統計數據 → 📈 matplotlib/plotly → 🖼️ 圖表檔案
```

**重點觀念**
- 圓餅圖 vs 長條圖 vs 折線圖的使用時機
- 視覺化的目的是「說故事」

---

### 🔷 階段 3：網頁儀表板

**學習目標**
- 理解前後端分離的概念
- 學會用 Streamlit 快速建立互動介面

**Vibe Coding 練習**
```
💬 給 AI 的 Prompt 範例：
「用 Streamlit 幫我做一個記帳儀表板：
1. 可以上傳 Excel 檔案
2. 自動顯示消費統計
3. 有互動式圖表（可以選擇月份）
4. 畫面要簡潔好看」
```

**系統架構圖**
```
👤 使用者上傳 → 🌐 Streamlit介面 → 🐍 Python處理 → 📊 即時圖表
```

## 🟡 專案二：天氣數據追蹤站

> **情境**：追蹤你關心城市的天氣，自動產生報表

### 🔷 階段 1：資料處理小工具

**學習目標**
- 理解 API 是什麼（「跟別人的電腦要資料」）
- 學會處理 JSON 格式資料

**Vibe Coding 練習**
```
💬 給 AI 的 Prompt 範例：
「幫我寫一個程式：
1. 用中央氣象署的 Open API 取得台北市天氣
2. 抓取溫度、降雨機率、天氣描述
3. 把這些資料存成 Excel
4. 我有 API Key 是 XXXXXX」
```

**資料流架構圖**
```
🌐 氣象API → 📥 JSON資料 → 🔄 解析轉換 → 📁 Excel儲存
```

**重點觀念**
- API = 資料的取貨口
- JSON = 資料的包裝格式
- 不用懂 HTTP 協定，只要會「描述要什麼資料」

---

### 🔷 階段 2：用 Python 畫圖

**Vibe Coding 練習**
```
💬 給 AI 的 Prompt 範例：
「我有一週的天氣資料 Excel，幫我：
1. 畫溫度變化折線圖
2. 畫降雨機率的面積圖
3. 把兩張圖放在同一個畫面上下排列
4. 輸出成 PNG 圖片」
```

**資料流架構圖**
```
📁 天氣Excel → 🐍 pandas讀取 → 📈 matplotlib繪圖 → 🖼️ PNG圖檔
```

---

### 🔷 階段 3：網頁儀表板

## 🔴 專案三：AI 智慧報表產生器

> **情境**：上傳任何 Excel，AI 自動分析並產生報告

### 🔷 階段 1：資料處理小工具

**學習目標**
- 理解如何讓 AI 分析資料內容
- 學會設計「通用型」的資料處理流程

**Vibe Coding 練習**
```
💬 給 AI 的 Prompt 範例：
「幫我寫一個程式：
1. 讀取任意 Excel 檔案
2. 自動判斷哪些欄位是數字、哪些是文字
3. 對數字欄位計算平均、最大、最小值
4. 對文字欄位計算出現次數
5. 輸出一份統計摘要」
```

**資料流架構圖**
```
📁 任意Excel → 🔍 欄位類型判斷 → 🤖 自動統計 → 📋 摘要報告
```

**重點觀念**
- 「通用化」思維：不寫死欄位名稱
- 資料型態決定處理方式

---

### 🔷 階段 2：用 Python 畫圖 + AI 解讀

**Vibe Coding 練習**
```
💬 給 AI 的 Prompt 範例：
「根據上一步的分析結果：
1. 自動選擇適合的圖表類型
2. 數字欄位畫長條圖比較
3. 類別欄位畫圓餅圖
4. 呼叫 OpenAI API，請 AI 用一段話解釋這份資料的重點」
```

**資料流架構圖**
```
📋 統計摘要 → 🤖 AI選擇圖表 → 📊 自動繪圖
                ↓
            🧠 GPT API → 📝 文字解讀
```

## 🎓 教學重點提醒

### ✅ 要強調的觀念

| 觀念 | 說明 |
|------|------|
| **Prompt 就是程式** | 描述越清楚，AI 給的程式越好用 |
| **資料流思維** | 永遠先想「資料從哪來 → 要怎麼處理 → 輸出到哪去」 |
| **分段驗證** | 每一步都先測試，確認對了再往下 |
| **不要怕錯誤** | 把錯誤訊息丟給 AI，讓它幫你修 |

### ❌ 不要花時間在

- 語法背誦
- 手動除錯（讓 AI 幫忙）
- 從零開始寫程式碼
